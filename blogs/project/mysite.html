    # 一、概述分析及流程

    学习地址：https://blog.csdn.net/chachapaofan/article/details/86571657 (opens new window)

    # 1.1、概述

    学习完成springboot不久希望做个项目练手，刚好个人博客网站开发技术栈相符且对本人后期学习有用，所以决定尝试学习开发这个个人博客网站

    # 1.2、分析

    • 系统主要完成以下几方面的功能：
    • 用户管理：用户的注册和登录，发表博文和评论。
    • 博文管理：用户可以在网站中发表和设置博文。
    • 评论管理：用户可以评论博文和回复其他用户的评论。
    • 分类管理：添加和删除分类，给文章设置分类。
    • 标签管理：添加和删除标签，给文章设置标签。

    # 1.3、系统功能

    # 1.3.1 用户管理

    用户的相关信息如下：用户ID、用户IP、用户名、用户昵称、用户密码、用户邮箱、用户头像、注册时间、用户生日、用户年龄、用户手机号。

    用户注册时需提供用户名、用户密码、用户邮箱或用户手机号。

    用户登录时可以采用用户名或邮箱或手机号进行登录。

    用户可以发布博文、发表评论、回复，还可以添加其他用户为好友。

    # 1.3.2 博文管理

    博文的相关信息如下：博文ID、发布日期、发表用户、博文标题、博文内容、点赞数、回复数、游览量。

    博文可以被普通用户发布、修改、删除和评论，但修改和删除仅限于自己发表的动态。

    博文发布时需要设置分类、标签。

    # 1.3.3 评论管理

    评论的相关信息如下：评论ID、评论日期、点赞数、发表用户、评论文章ID、评论内容、父评论ID。

    评论可以被用户发表和删除以及被其他用户回复。

    # 1.3.4 分类管理

    分类的相关信息如下：分类ID、分类名称、分类别名、分类描述、父分类ID。

    只有管理员可以添加、删除、修改分类。

    分类的作用不仅可以将文章分类，还可以作为博客的菜单。

    # 1.3.5 标签管理

    标签的相关信息如下：标签ID、标签名称、标签别名、标签描述。

    用户发表文章时可以设置标签，标签不仅可以将文章分类，还可以作为博客的菜单。

    # 3.1、遇到的问题

    pojo层:

    • 千万记得数据库字段名映射到pojo类需要改成驼峰命名!!!

    • 数据库中的datetime类型映射到pojo https://blog.csdn.net/liubin5620/article/details/80309518

    • 数据库中的context类型映射到pojo为String

    dao层:

    • @Mapper说明这个类是一个mybatis的mapper的接口
    • @Respostory为在Spring容器中注册dao层组件
    • 记得更换mapper.xml的namespace对应的是dao层!!!

    service层:

    • Impl中记得加上注解@Service和类内的@Autowired

    • 接口中记得把原来的返回值按规律改变
